# 地图可视化

当用户需要以下操作时，使用此技能：

- 在地图上用标记显示位置。
- 可视化点与点之间的路线或路径。
- 在地图上勾勒区域或边界。
- 在一张地图上组合标记、路线和区域。
- 将结构化位置数据转化为渲染地图块。

此技能使用围栏代码块为**应用内消息渲染器**（聊天消息列表 UI）输出地图 Markdown：

````markdown
```map
{ ...json... }
```
````

## 核心规则

- 始终在 ` ```map ` 块内输出有效的 JSON。
- 将数字字段保持为数字，而非字符串。
- 每个坐标使用 `lat` 和 `lon`。
- 每张地图聚焦于一个空间问题。
- 若数据不完整或有歧义，在地图前简要说明假设。

## 运行时目标

- 目标渲染器：任何支持 ` ```map ` 围栏块的聊天/消息渲染器。
- 预期格式：带有语言标签 `map` 的围栏代码块和一个 JSON 对象。
- 若不确定，仍然输出确切的 ` ```map + JSON ` 结构。

## 支持的字段

- `title`
- `height`
- `center`
- `span`
- `markers`
- `polylines`
- `polygons`

## JSON 模式

```json
{
  "title": "SF Delivery Route",
  "height": 260,
  "center": { "lat": 37.7749, "lon": -122.4194 },
  "span": 0.05,
  "markers": [
    {
      "lat": 37.7749,
      "lon": -122.4194,
      "title": "Warehouse",
      "tint": "blue"
    },
    {
      "lat": 37.7849,
      "lon": -122.4094,
      "title": "Dropoff",
      "tint": "#FF5A36"
    }
  ],
  "polylines": [
    {
      "coordinates": [
        { "lat": 37.7749, "lon": -122.4194 },
        { "lat": 37.7798, "lon": -122.4140 },
        { "lat": 37.7849, "lon": -122.4094 }
      ],
      "color": "blue"
    }
  ],
  "polygons": [
    {
      "coordinates": [
        { "lat": 37.7700, "lon": -122.4300 },
        { "lat": 37.7800, "lon": -122.4300 },
        { "lat": 37.7800, "lon": -122.4100 },
        { "lat": 37.7700, "lon": -122.4100 }
      ],
      "fillColor": "#22AA66",
      "strokeColor": "green"
    }
  ]
}
```

## 字段参考

### `title`

- 可选字符串，显示在地图卡片上方。

### `height`

- 可选数字。
- 运行时默认值：`240`。
- 运行时限制范围：`160...360`。

### `center`

```json
{ "lat": 37.7749, "lon": -122.4194 }
```

- 可选对象。
- 若存在，`lat` 和 `lon` 必须是有效坐标。
- 若省略 `center`，渲染器将尝试将可视区域适配到标记、折线和多边形中所有有效坐标。

### `span`

- 可选数字，控制缩放/区域大小。
- 运行时默认值：`0.05`。
- 运行时限制范围：`0.001...120`。

### `markers`

```json
[
  {
    "lat": 37.7749,
    "lon": -122.4194,
    "title": "Warehouse",
    "tint": "blue"
  }
]
```

- 可选数组。
- 每个标记支持：
  - `lat`
  - `lon`
  - `title?`
  - `tint?`

### `polylines`

```json
[
  {
    "coordinates": [
      { "lat": 37.7749, "lon": -122.4194 },
      { "lat": 37.7849, "lon": -122.4094 }
    ],
    "color": "blue"
  }
]
```

- 可选数组。
- 每条折线支持：
  - `coordinates`
  - `color?`
- 折线仅在包含至少 2 个有效坐标时才渲染。

### `polygons`

```json
[
  {
    "coordinates": [
      { "lat": 37.7700, "lon": -122.4300 },
      { "lat": 37.7800, "lon": -122.4300 },
      { "lat": 37.7800, "lon": -122.4100 }
    ],
    "fillColor": "#22AA66",
    "strokeColor": "green"
  }
]
```

- 可选数组。
- 每个多边形支持：
  - `coordinates`
  - `fillColor?`
  - `strokeColor?`
- 多边形仅在包含至少 3 个有效坐标时才渲染。

## 坐标规则

- 纬度必须在 `-90` 到 `90` 之间。
- 经度必须在 `-180` 到 `180` 之间。
- 无效坐标会被渲染器忽略。

## 有效性规则

地图块在满足以下至少一个条件时有效：

- 有一个有效的 `center`
- 至少有一个有效的标记
- 至少有一条包含 2 个或更多有效坐标的折线
- 至少有一个包含 3 个或更多有效坐标的多边形

若以上条件均不满足，则不应输出地图。

## 颜色规则

颜色字段支持：

- 命名颜色，如 `red`、`orange`、`yellow`、`green`、`mint`、`teal`、`cyan`、`blue`、`indigo`、`purple`、`pink`、`brown`、`gray`、`black`、`white`
- 十六进制颜色，如 `#FF5A36`
- 运行时也接受 8 位十六进制颜色

## 元素选择指南

- 使用 `markers` 表示命名地点或停靠点。
- 使用 `polylines` 表示路线、轨迹或有序移动。
- 使用 `polygons` 表示区域、服务范围或边界。
- 当需要明确框架时，使用 `center` + `span`。
- 当希望渲染器自动适配可视区域时，省略 `center`。

## 工作流程

1. 识别空间问题：地点、路线或区域。
2. 将所有坐标规范化为 `lat` 和 `lon`。
3. 选择所需的最小元素集：标记、折线、多边形。
4. 仅在明确框架有帮助时才添加 `center` 和 `span`。
5. 每张地图生成一个 ` ```map ` 块。
6. 可选：在地图后添加简短洞察。

## 质量标准

- 不编造坐标。
- 不输出格式错误的 JSON。
- 标题简短且具体。
- 坐标不使用字符串类型。
- 确保每条折线至少有 2 个有效点。
- 确保每个多边形至少有 3 个有效点。
- 保持每张地图聚焦且易读。
