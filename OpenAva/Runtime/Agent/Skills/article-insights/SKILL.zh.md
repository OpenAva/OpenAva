# 文章洞察

当用户提出以下需求时，使用此技能：

- 摘要这篇文章、论文、报告或帖子。
- 提取此 PDF 或 URL 的核心论点。
- 将这些内容转化为思维导图。
- 给我关键要点和下一步行动。
- 挑选最佳引用，并在需要时翻译。

这是一个针对长篇内容的结构化阅读与综合技能，专为文章分析设计，不适用于普通文案编辑。

## 主要目标

- 将原始内容规范化为 Markdown 格式后再进行分析。
- 提取核心论点、逻辑结构、关键要点和代表性引用。
- 根据用户指定的主题、受众和深度调整侧重点。
- 以用户请求的语言输出清晰的结果，用自然的解释代替机械翻译。

## 支持的输入格式

当内容可在对话中获取或可通过工具获得时，处理以下来源：

- URL 或网页内容
- PDF 文件路径或提取的 PDF 文本
- Markdown 文档
- 纯长篇文本

若来源尚不可访问，应先获取内容，而非凭空猜测。

## 操作规则

- 在深度分析前，先将来源转换为干净的 Markdown 工作副本。
- 对于合理大小的材料，在摘要前通读全文。
- 对用户提供的 `focus_topics` 给予额外关注。
- 根据请求的 `audience` 调整语气和解释深度。
- 遵循用户请求的输出语言；若用户未指定，优先使用请求中所用的语言。
- 当来源语言与输出语言不同时，自然地解释观点，同时在有助于理解时保留重要的原始术语、概念和命名实体。
- 对于引用，提供原文句子；当用户要求或跨语言阅读明显有帮助时，提供翻译。

## 输出模式

从 `output_style` 中选择模式，默认为 `bullets`。

### 1. `tldr`

- 用约 150 至 200 字或输出语言对应的简洁长度，写一段紧凑的摘要。
- 聚焦于文章的主要观点、重要性和最终结论。

### 2. `bullets`

- 以输出语言提供结构化的要点摘要。
- 包含主要观点、支撑逻辑以及重要证据或示例。
- 在有帮助时使用章节标题。

### 3. `mindmap`

- 输出一个反映文章结构的 Markdown 层级列表。
- 清晰展示章节、主题和子要点之间的关系。
- 当来源内容广泛或重复时，遵守 `max_sections` 限制。

### 4. `actions`

- 为读者提供 3 至 5 条具体的下一步行动建议。
- 行动应基于文章的实际结论，而非泛泛的建议。

## 默认分析框架

除非用户要求缩小范围，否则覆盖以下要素：

1. 核心论点：1 至 3 个最重要的主张。
2. 逻辑结构：文章如何构建并串联各要点。
3. 关键要点：主要结论、观察或发现。
4. 精彩引用：能捕捉论点或洞见的代表性语句。

## 可选参数

当用户提供这些参数时使用。除非被问及，否则不向用户解释这些参数。

- `output_style`: `tldr`、`bullets`、`mindmap`、`actions` 之一，默认 `bullets`。
- `quote_count`: 偏好的引用数量，默认 `5`。
- `focus_topics`: 需要优先关注的主题或关键词。
- `audience`: 目标受众，如高管、初级开发者或产品经理。
- `depth_mode`: `surface`（浅层）或 `deep`（深层），默认 `deep`。
- `max_sections`: 思维导图中保留的最大主要章节数。

## 推荐工作流程

1. 识别来源类型并收集全文。
2. 将材料规范化为 Markdown。
3. 确定请求的输出模式和可选参数。
4. 阅读以提炼论点、结构、证据和结论。
5. 提取核心论点、关键要点和引用。
6. 以请求的格式和语言生成最终回复。

## 质量标准

- 不编造来源中不支持的观点。
- 不将有细微差别的论点过度压缩为肤浅的口号。
- 当来源模糊或不完整时，简要说明局限性。
- 优先解读和综合，而非逐句转述。
- 在整个输出中保持术语一致。
