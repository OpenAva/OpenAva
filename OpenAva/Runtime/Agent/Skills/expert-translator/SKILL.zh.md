# 专业翻译

当翻译任务要求精确度、自然措辞、风格控制和格式完整性时，使用此技能。

## 输入

- `source_text`: 需要翻译的原文。
- `target_language`: 目标语言，例如 `Chinese`（中文）、`English (US)`（美式英语）或 `English (UK)`（英式英语）。
- `style_and_tone`: 目标风格，例如 `Academic`（学术）、`Business Professional`（商务专业）、`Casual and Humorous`（轻松幽默）或 `Literary and Elegant`（文学优美）。
- `glossary`（可选）: 必须一致翻译的术语键值对映射。

## 工作流程

1. 分析请求，识别目标语言、所需语气、领域词汇和任何术语表约束。
2. 产出完全忠实于原文的初稿翻译。
3. 对照原文审查草稿，检查准确性、遗漏、添加内容、流畅度、术语一致性和风格对齐。
4. 精炼翻译，使其读起来像目标语言的原创写作，同时保留原文含义。
5. 仅返回最终润色后的翻译，除非用户明确要求提供辅助说明。

## 翻译规则

- 精确保留原始格式，包括 Markdown、列表、表格、链接、强调、LaTeX、代码围栏和 YAML frontmatter。
- 不翻译锁定区域，如代码块、行内代码、URL 和 YAML frontmatter 的键。
- 出现时精确应用术语表中的术语。
- 先注重语义忠实，再在精炼阶段改善措辞。

## 审查清单

- 准确性：无误译、遗漏或添加含义。
- 流畅性：无生硬的直译或翻译腔。
- 风格：符合所请求的受众和语气。
- 术语：术语表使用正确且一致。
- 格式完整性：所有格式和锁定区域保持完整。

## 输出规则

- 默认输出仅为最终翻译文本。
- 不暴露内部推理、逐步分析或审查备注。
- 若用户明确要求术语表使用说明或对比列表，在翻译后附加一个简短的明确标注章节。

## 质量标准

- 最终翻译应感觉母语化、地道，并在风格上与目标语言对齐。
- 精炼可以大胆，但必须忠实于原文意图。
