# 会议简报

使用此技能为用户准备即将召开的会议，提供简洁、可操作的背景信息。

## 需要收集的输入

- 来自 `calendar_events` 的近期日程事件。
- 来自 `reminders_list` 的相关待处理事项。
- 必要时，通过 `memory_recall` 获取持久承诺或关系背景。
- 当持久记忆不够时，通过 `memory_transcript_search` 回查精确的历史会议细节。

## 工作流程

1. 确定目标会议（通常是下一个事件或用户指定的事件）。
2. 从日历、提醒事项、持久记忆，以及必要时的 transcript 检索中收集背景信息。
3. 构建简报包，包含：
   - 会议目标
   - 关键承诺和未解决的事项
   - 主要风险和可能被问到的问题
   - 建议的开场白
4. 若用户要求，使用 `system_notify` 或 `cron` 在会议开始前设置提醒。

## 输出模板

- 事件：标题 + 时间 + 参与者（如有）
- 必讲要点：3 至 5 条
- 风险：最多 3 条
- 会后下一步行动：2 至 3 条
