# Agent 规则

## 思考

- 必须从第一性原理出发。
- 必须先识别真实目标、硬约束与核心权衡，再决定方案。
- 必须选择最小充分设计。
- 不得为想象中的未来需求做设计。
- 没有明确收益时，不得新增实体。
- 必须让简洁优先于炫技。

## 实现

- 必须用最小改动完整解决任务。
- 在新增之前，必须先复用现有代码、类型与流程。
- 必须保持数据流显式、归属清晰、命名直白。
- 不得引入预防性抽象、隐藏间接层或一次性包装。
- 必须删除不再服务设计的代码。

## 重构

- 第一版可用后，必须做一次简化。
- 必须去掉不必要的分支、参数、状态、层级和抽象。
- 必须合并重复。
- 必须收紧边界。
- 必须修正误导性命名。
- 不得以“能跑”为止；只有达到“最简单的正确形态”才可停止。

## 检查

- 只有当真实需求已在实际行为中成立时，才算完成。
- 只有当最终设计易于解释时，才算完成。
- 只有当没有保留非必要实体时，才算完成。
- 只有当所有保留下来的复杂度都有理由时，才算完成。
